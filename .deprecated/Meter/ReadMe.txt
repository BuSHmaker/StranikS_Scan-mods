  Назначение
  ==========

Мод предназначен и распространяется исключительно для исследования игры,
анализа элементов её механики и оценки значений различных игровых параметров.

  Установка
  =========

Для установки мода достаточно извлечь папки из архива в каталог игры World_of_Tanks\res_mods\0.X.X

  Текстовые метки и картинки на экране
  ====================================

Мод позволяет вывести на экран  в реальном времени значения различных игровых параметров, в первую очередь связанных с механикой игры, в виде текстовых меток и картинок. Список параметров, доступных для отображения, представлен в файле настроек "Meter.cfg" в секции "TextLabels". Там же можно задать координаты каждой метки, отключить или включить её отображение, а также настроить цвет и формат представления чисел. Помимо текстовой информации могут отображаться и различные картинки, например иконки оборудования, установленного на танке игрока и на танках союзников и противников.

  Коллинжи бронирования танков
  ============================

Ввиду того, что механика игры, а также сервер, оперируют только коллинжами пробития, например при проверке обнаружения танка по засвету, при расчете попаданий и пробитий и т.д., то в моде добавлена возможность отображения коллинжей пробития вместо стандартных моделей танков. По умолчанию коллинжи отключены, если их включить в файле настроек "Meter.cfg" в секции "CollisionSkins", то при наведении прицела на танк союзника или противника стандартная модель танка заменяется на модель пробития.

  Вывод значений в лог
  ====================

Данные выводятся в лог в соответствии с маской, которая прописана в файле-форматтере. По умолчанию это файл "Log.cfg". Он содержит пример вывода всех параметров, которые на данный момент доступны в моде, через макросы подстановки значений. Рассмотрим пример вывода в лог значений координат попадания снарядов игрока во время боя. Для этого создадим текстовый файл, например "hitLog.cfg" в папке с модом и пропишем в нем две строчки:

{%ShotCount%} {%HitPointPosition%}

  
Вторую строку оставим пустой, чтобы значения записывались в столбик. Затем укажем имя файла в качестве нового форматтера лога в файле настроек "Meter.cfg":
  
"Log" -> "Formatter": "hitLog.cfg"
  
Запускаем клиент игры. После чего, зайдя в бой или находясь в ангаре, включаем запись лога комбинацией клавиш ALT+NUMPAD1. По умолчанию ведение лога выключено. В бою при каждом выстреле и попадании снаряда игрока во что-либо видимое в файле лога logs\log_210415_003507_753.log будут появляться записи их координат:

1 253.5314483643 0.2238807678 -269.8118591309
2 259.2808837891 -0.3629567623 -268.1839599609
3 248.8338012695 -0.0172061920 -208.1577453613
...

Параметры ведения лога можно настроить в файле "Meter.cfg", там задаются: комбинации горячих клавиш, начальное состояние, время обновления, местоположение файлов и т.д. При необходимости формирования логов разного формата рекомендуется создавать отдельные cfg-файлы форматтера лога и подключать их в настройках мода.

Если требуется чтобы запись в лог производилась только когда меняется какой-то конкретный параметр или группа параметров, то их можно пересчислить в свойстве "Conditions" в файле "Meter.cfg". Например, если вписать туда ["%ShotCount%"] и запустить игру, то запись в лог будет производится только в моменты выстрела игроком.

  Дамп значений в файл
  ====================

Предназначен для однократной записи в файл значений параметров по нажатию комбинации клавиш, по умолчанию ALT+NUMPAD0. В отличие от лога дамп каждый раз записывается в новый файл. Данный инструмент реализован и работает отдельно от лога и имеет свою секцию настроек в файле "Meter.cfg". Как и в случае с логом маска вывода дампа задается пользователем в отдельном файле-форматтере, по умолчанию "Dump.cfg". Все макросы доступные в форматтере лога поддерживаются и дампом. Настройки дампа аналогичны настройкам лога.

  Шрифт текстовых меток
  =====================

В игре настройка шрифта экранных меток осуществляется посредством текстовых font-файлов. Для данного мода это файлы "system\fonts\Meter.font" и "system\fonts\ZoomIndicator.font".

  Горячие клавиши
  ===============

Комбинации клавиш работают как в бою так в ангаре, например вы можете включить/отключить лог находясь в ангаре до захода в бой. Если в бою сообщения мода появляются внизу экрана, то в ангаре они выводятся в центре уведомлений.

  Обновление значений и нагрузка
  ==============================

Мод реализован таким образом чтобы обеспечивалась возможность вывода значений с высокой точностью на малых интервалах обновления без существенного падения производительности игры. Значения параметров на экране обновляются только для видимых текстовых меток. Кэширование значений для записи лога и дампа происходит только для тех параметров, макросы которых прописаны пользователем в их форматтерах. Не зависимо от интервала обновления лога, указанного в его настройках, запись в файл производится только если меняется какой-либо из отслеживаемых логом параметров. Не смотря на это, при малых интервалах обновления мод может стать ощутимой нагрузкой для клиента, в связи с чем рекомендуется использовать его исключительно в тех целях, для которых он предназначен.
